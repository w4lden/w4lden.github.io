5、重要说明

当有多个账户，特别是之前设置了全局用户名和邮箱的情况下，需要unset一下。当某一个仓库没有设置自己的user.name和user.email时，使用的就是全局信息。相反，当仓库下的config有设置单独的user.name和user.email时，则使用当前信息。如果没有注意全局用户名和邮箱这个问题的话，很有可能“系统”根据全局的用户名和邮箱信息更新提交者信息，这样在我们的目标仓库中就会出现全局用户的贡献者记录，这是我们不愿意看到的。（如果全局邮箱是假的，则只会显示全局用户名称，点击名字后，并不会跳转到该用户的github下，这部分知识主要参考了参考链接5和6）

unset全局信息

也可以不取消git的全局用户名和邮箱。只需要在另一个博客（不同的仓库下，比如在公司的repository下）的git目录下设置局部的用户名和邮箱即可。

在自己的github的仓库再执行刚刚的命令一遍即可,这样就可以在不同的仓库，以不同的账号登录。
